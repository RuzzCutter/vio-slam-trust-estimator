## Гипотеза

Адаптивная оценка достоверности визуальных измерений `c(t)` снижает навигационную ошибку и повышает устойчивость локализации по сравнению с классическим VIO при **деградации** визуального потока. На **чистых** данных adaptive не должен ухудшать точность:

```text
ATE_adaptive  ≈  ATE_baseline   (допуск: ATE_adaptive <= 1.05 * ATE_baseline)
```

---

## Принятые решения

1. **Сравнение:** только OpenVINS baseline vs OpenVINS+adaptive (VINS-Mono/ORB-SLAM3 — литобзор).
2. **Карта достоверности:** входит в первую публикацию, реализуется после адаптивной ковариации.
3. **Данные:** EuRoC + TUM-VI; натурные эксперименты — только описание в тексте.
4. **Диск:** ~57 ГБ свободно; minimal EuRoC скачан (MH_01, MH_04, MH_05).
5. **Срок:** упрощать протокол; часть результатов допустимо экстраполировать.

## Математическое ядро эксперимента

### 1. Состояние и фильтр (OpenVINS MSCKF)

В каждый момент времени `t` фильтр оценивает состояние `x_t` (поза IMU, скорость, смещения, клоны поз в скользящем окне, параметры SLAM-точек). Прогноз — по IMU; визуальные измерения MSCKF вносят коррекцию через EKF-update.


| Режим        | Шум измерений               |
| ------------ | --------------------------- |
| **Baseline** | фиксированный `R0`          |
| **Adaptive** | масштабируется через `c(t)` |


---

### 2. Визуальное измерение (упрощённая модель)

Для 3D-точки `i` и камеры `c`:

```text
z[i,c] = h(x, f[i]) + v[i,c]

v[i,c] ~ N(0, sigma_pix^2 * I_2)
```


| Символ          | Смысл                                                        |
| --------------- | ------------------------------------------------------------ |
| `z[i,c]`        | измеренные пиксельные координаты (R^2)                       |
| `h(.)`          | модель проекции (калибровка + поза камеры)                   |
| `f[i]`          | параметры i-й 3D-точки                                       |
| `sigma_pix`     | базовый шум пикселя (`up_msckf_sigma_px`, по умолчанию 1 px) |
| `sigma_pix_0^2` | опорная дисперсия в baseline                                 |


Невязка (residual) для inlier-набора:

```text
r = z - h(x_hat)     // r in R^(2N)
H = d h / d x_aug    // якобиан
```

---

### 3. Baseline: фиксированная ковариация

```text
R0 = sigma_pix_0^2 * I_(2N)

Omega_0 = R0^(-1) = sigma_pix_0^(-2) * I     // постоянная информационная матрица
```

---

### 4. Признаки достоверности f1 … f4

На каждом MSCKF-update: `f_k in [0, 1]` (`TrustEstimator::compute`).

#### f1 — доля inlier-ов (геометрия трекинга)

```text
f1 = N_inliers / N_features
```


| Переменная   | Описание                                      |
| ------------ | --------------------------------------------- |
| `N_features` | число отслеживаемых особых точек              |
| `N_inliers`  | число точек после gating (chi2-тест OpenVINS) |


При деградации изображения: `f1` падает.

#### f2 — репроекционная ошибка

```text
e_repr_mean = (1 / N_inliers) * sum_i ||r_i||_2     // по inlier-ам

f2 = exp(-e_repr_mean / 2)
```


| Переменная    | Описание                               |
| ------------- | -------------------------------------- |
| `e_repr_mean` | средняя L2-норма пиксельной невязки    |
| делитель `2`  | эмпирический масштаб (px), зашит в код |


Большие невязки -> `f2 -> 0`.

#### f3 — пространственное покрытие изображения

Сетка `G x G` (`trust_grid_size`, по умолчанию G=8):

```text
f3 = (число заполненных ячеек) / G^2

ячейка (u,v) заполнена, если в ней есть >= 1 inlier
```

Координаты нормируются на размер кадра (752 x 480 для EuRoC). Кластеризация точек -> `f3` падает.

#### f4 — неопределённость позы (вероятностный признак)

```text
f4 = exp(-tr(P_pos) / tau)

tr(P_pos) = P_xx + P_yy + P_zz     // только блок позиции 3x3
```


| Переменная | Описание                                      |
| ---------- | --------------------------------------------- |
| `P_pos`    | 3x3 блок позиции из ковариации клона IMU-позы |
| `tau`      | `trust_tau`, по умолчанию 5.0                 |


> **Важно:** не использовать полный trace 6x6 — дисперсия ориентации обнуляет f4 на длинных траекториях (fix 2026-06-18).

---

### 5. Интегральный коэффициент доверия c(t)

```text
c(t) = clip( (sum of active f_k) / n_active ,  c_min,  c_max )

n_active = 4 (full) или 3 (ablation: один f_k отключён через trust_use_f*)
c_min = 0.05 ,  c_max = 1.0     // зашиты в код
```

`c(t)` — скалярная оценка качества визуального update в момент `t`.

---

### 6. Адаптивная ковариация измерений

```text
s(t) = min( 1 / (c(t) + eps) ,  s_max )

sigma_pix_eff^2(t) = sigma_pix_0^2 * s(t)
                   = sigma_pix_0^2 / (c(t) + eps)     // с ограничением s_max

R_t = sigma_pix_eff^2(t) * I_(2N)
```


| Параметр | YAML-ключ                | По умолчанию |
| -------- | ------------------------ | ------------ |
| `eps`    | `trust_eps`              | 0.05         |
| `s_max`  | `trust_r_max_multiplier` | 100          |


**Смысл:** низкий `c(t)` -> больший шум -> меньший вес vision в EKF -> фильтр сильнее опирается на IMU.

Информационная формулировка:

```text
Omega_t = R_t^(-1)  ~  (c(t) + eps) / sigma_pix_0^2   ~  c(t)
```

---

### 7. Отказ от визуального update

```text
если  c(t) < c_skip  ->  пропустить EKFUpdate (только IMU)

c_skip = 0.15   (trust_skip_threshold)
```

---

### 8. EKF-update (MSCKF)

```text
S = H * P * H^T + R_t
K = P * H^T * S^(-1)

x_hat+ = x_hat- + K * r
P+     = (I - K * H) * P-
```


| Символ | Описание                        |
| ------ | ------------------------------- |
| `P`    | априорная ковариация            |
| `H`    | якобиан stacked residuals       |
| `r`    | вектор невязок                  |
| `R_t`  | adaptive или фиксированный `R0` |


**Код:** `UpdaterMSCKF.cpp` — `sigma_pix_sq_effective` перед `R_big`.

### 9. Метрики эксперимента

#### ATE (Absolute Trajectory Error)

После Sim(3)-выравнивания (`evo --align --correct_scale`):

```text
ATE_RMS = sqrt( (1/N) * sum_k || p_k_est - p_k_gt ||_2^2 )
```


| Символ    | Описание                                 |
| --------- | ---------------------------------------- |
| `p_k_est` | оценённая позиция (`trajectory_tum.txt`) |
| `p_k_gt`  | ground truth (MH_04, MH_05)              |
| `N`       | число синхронизированных пар             |


#### RPE (Relative Pose Error)

```text
RPE_RMS = sqrt( (1/M) * sum_j || delta_p_j ||_2^2 )     // delta = 1 m
```

#### Дополнительные метрики


| Метрика          | Определение                                                            |
| ---------------- | ---------------------------------------------------------------------- |
| Tracking failure | разрыв > 0.5 с, `[init]: failed`, или `diverged=1` (max|coord| > 50 m) |
| `c_mean`         | среднее `c(t)` по `trust_log.csv`                                      |
| CPU/RAM          | `/usr/bin/time -v`                                                     |

### 10. Модели деградации

Только изображения в bag; IMU не меняется.

#### D1 — Gaussian blur

```text
I'(x,y) = (G_sigma * I)(x,y)
```


| Параметр demo | sigma |
| ------------- | ----- |
| `gaussian:5`  | 5 px  |
| `gaussian:9`  | 9 px  |


#### D3 — засветка

```text
I'(x,y) = clip( I(x,y) * (1 + p/100),  0, 255 )
```


| Параметр demo   | p    |
| --------------- | ---- |
| `brightness:40` | +40% |
| `brightness:60` | +60% |


**Ожидаемая цепочка:** sigma или p растёт -> f1,f2,f3 падают -> c(t) падает -> adaptive ослабляет vision.

---

### 11. Критерии подтверждения гипотезы

```text
1. Деградация:  ATE_adaptive < 0.9 * ATE_baseline   (>= 2 сценария)
2. Failures:    N_fail_adaptive <= N_fail_baseline
3. Чистые:      ATE_adaptive <= 1.05 * ATE_baseline
4. Монотонность: c_mean(D[i+1]) < c_mean(D[i]) при усилении деградации
```

---

### 12. Карта достоверности

```text
C(p) = sum_t w_t * c(t) / sum_t w_t
w_t = exp( -||p_hat_t - p||^2 / (2*r^2) )
```

Упрощение: 2D-сетка в плоскости xy; штраф в планировщике при `C(p) < c_th`.
